# CLAUDE.md

## 核心规则

- 只做本次任务相关的最小修改
- 未经用户明确要求，不运行测试/构建/迁移/部署
- 未经用户明确要求，不启动或重启服务
- 不修改密钥、数据库、运行时产物
- 不做无关重构
- 不要一上来全盘扫描项目

## 默认读取

普通任务默认先读：

1. `PROJECT_MAP.md`
2. `TASK.md`

`TASK.md` 为空时，以用户当前消息作为本次任务。

## 子目录文档

- 先根据任务定位目标目录
- 只读目标目录相关的 `README.md` / `CLAUDE.md` / `AGENTS.md`
- 不要全局读取所有 README
- 文档和代码冲突时，以当前代码为准

## 高风险任务

以下情况必须读取目标子项目文档，并用 `rg` 核实入口、真实路径和调用链：

- 跨子项目
- `PROJECT_MAP.md` 标注为待确认
- 用户要求只分析原因
- 涉及入口、调用链、发布、上传、数据库、配置

只分析任务不得修改代码。

## 回复要求

完成后只回复：

- 改了哪些文件
- 主要改了什么
- 未执行哪些验证或操作

如果是只分析任务，必须说明依据来自哪些文件或 `rg` 结果。
