# AI Toolbox — AI 短视频广告全链条生成

为 S101 苦荞精酿啤酒自动生成短视频广告素材的多项目仓库。

## 子项目

| 目录 | 用途 | 技术栈 | 端口 |
|------|------|--------|------|
| [`work/`](work/) | **主项目** — 8 步 Workflow 流水线 | FastAPI + React 19 + SQLite | 后端 :8000 / 前端 :5173 |
| [`alxuanchuan/`](alxuanchuan/) | 同功能 Gemini 变体 | Express + React | :3000 |

## 快速启动

```bash
# 主项目后端
cd work && python server.py          # → http://localhost:8000

# 主项目前端
cd work && npm run dev               # → http://localhost:5173

# Gemini 变体
cd alxuanchuan && npm run dev        # → http://localhost:3000
```

# 产品需求文档（原始）

> 以下为项目启动时的产品级需求描述，反映原始设计意图。当前实现与原始需求存在差异，以实际代码为准。

我们想制作的视频模板跟"1.53 :4pm 11/26 a@N.ws Njp:/ 青柠果香中和了啤酒苦涩，喝起来清爽过瘾！# 啤酒 # 饮料推荐 # 聚会必备 # 果酒  https://v.douyin.com/b9xiPL1Z4ZQ/ 复制此链接，打开Dou音搜索，直接观看视频！"，"4.38 :0pm reo:/ 10/31 f@o.DU 全开盖这样买才划算  https://v.douyin.com/h-TGc-0K9wk/ 复制此链接，打开Dou音搜索，直接观看视频！"这两个链接中的视频差不多，无人出镜，有文案，产品，这是真人实拍，是正常性，而我们要达到的效果跟这个一样，gpt-image-2达到真实效果，视频模型把这个图作为底子，生成的视频会具有真实性，因为没有真人出镜。
还可以这种脑洞打开具有趣味性剧情（前提是环境内必须有我们的产品），没有文案那种：0.00 :0pm bnQ:/ m@q.Eh 09/24 四瓶以后，我那可爱奔放的第二人格才会显现。# 微醺快乐水 # 微醺的快乐 # 喝酒 # 上热门话题喝酒  https://v.douyin.com/uzJDY53ICGw/ 复制此链接，打开Dou音搜索，直接观看视频！
还可以这种休闲性："5.33 :8pm xSL:/ s@R.Kw 03/01 亏我没有对象，不然这么热的天还得出去约会， 真的是太可怜了…# 微醺时刻 # 约会  https://v.douyin.com/kO-PIvKAmj8/ 复制此链接，打开Dou音搜索，直接观看视频！"，边看电视边喝我们的产品那种，各种趣味有意义的场景具有观赏性的剧情环境场景等都得随机生成提示词，然后自动分离出x关键帧，然后把这个x个关键帧的图片实现描述成提示词，格式："帮我生成n张图片：第1张：......第n张：......，注意，n张图的画质必须是由中低像素的手机刚拍摄的有种真实细微模糊感，不要有任何ai生成瑕疵"然后把这个提示词给gpt-image-2，图片生成好了之后，模型进行识别分析图片，来分析所有元素，按照自己生成的提示词和剧情编排排版，来评分这个图片（达到预期效果的程度：超低，低，中，高，超高），低和超低提示我们重新生成图片（确认重新生成则直接重新生成，选择继续直接继续这个效果下一步），中以及以上的程度直接开始下一步，分析产品参数生成文案。
最终，生成用于生成视频素材的总提示词，这个提示词参考图片分析结果，文案等。
比如：
《帮我生成一个视频：
（1）素材图解析：比如：图1是：.......图n是：......
（2）排版编排剧情：放第一步就生成的排版剧情原文。（比如这x个素材图要怎么怎么编排连贯转场怎么剧情发展怎么描述什么地方什么什么 （用到图片时候可艾特对应图，比如@图1中冰柜打开.....等等所有剧情编排排版剧情内容等都包含的排版编排剧情原文）
（3）文案诵读：比如：第一秒开始（男，女随机），开始人声：（这里放入刚生成的文案原文本，比如："有没有喝啤酒第二天头疼的？那是工业勾兑酒！试试这款S101苦荞精酿，纯粮发酵28天，不像普通啤酒七天就兑水上市。入口绵柔，回口荞香，好喝不上头。主页有链接。"最后一秒人声结束）》

然后把这个最后的总提示词给视频生成模型，开始生成视频。

工作介绍：

（可多选：kimi，gpt，Seedance，glm等等任何模型，需要效果最好的，等你推荐，或者我们第一步编排排版剧情提示词生成用一个模型，按照这个提示词进行分镜运镜啥的打包成用于gpt-image-2生图的模型用另一个模型，然后生图用image-2模型，后面的图片分析测评用另外一个模型（按照第一步的编排排版剧情提示词原文测评，然后按照公司参考数据生成文案的模型再用别的模型，最后的生成总提示词（用于视频生成的提示词）的模型再用另外的模型，加上所有的模型（生图和生视频也包含）总共6个模型，或者我们砍半，使用三个模型，固定的gpt-image-2和生成视频模型，还有总"导演"模型，如果使用三个模型，请你在总"导演"模型设置清除上下文等功能，避免上下文慢了越来越慢，出现幻觉，在一批任务中（第一步到最后一步），清除一次上下文即可，如果使用多个也就是6个模型，请你也合理加上释放措施，但是生成结果时，检查数据库历史记录对比相似度还是得正常运作，然后使用的模型按照最高理解能力（模型界的文科状元）就是表达能力理解能力多模态最强等筛选你定模型，然后，分析我们这个视频量可以用什么视频生成模型（即梦有点贵，你也可以推荐性价比更好的模型，因为我们不出镜，对人脸，人声等要求不是跟做大型商用动漫或者ai视频那么大，平常性价比最高的模型你来推荐）。）


第一步：ai自动发挥，围绕目标产品S101苦荞精酿啤酒【按照剧情筛选（趣味性，休闲性，正常性），角色筛选（男女随机，不露脸，只需要第一人称镜头，男女可区分于美甲或者手型等等，这里确定了性别之后，整个流程只能用选择的性别），场景筛选（覆盖餐厅，酒店，烧烤店，户外沙漠滩，海边，室内.....等等50种随机场景），图片风格：必须是低中像素的手机刚拍摄的细微模糊真实感】进行排版编排生成整个剧情，分镜运镜镜头语言等等。（切记，剧情等等都有相似度限制，这次的剧情排版编排等等不能跟上次等所有记录内容相似度不能高于60%。）
第二步：ai分离出关键帧每一个分镜，详细描述所有分镜以后，把这个描述打包成格式的提示词（提示词内需要提到我上传的目标产品图（.png或者称呼图片即可），因为gpt-image-2需要参考真实的商品正视图生成所有图片素材，不是凭空生成目标产品图），格式："帮我生成n张图片：第一张：......第n张：......，注意，n张图的画质必须是由中低像素的手机刚拍摄的有种真实细微模糊感，不要有任何ai生成瑕疵"。（注意，n≤12）
第三步：把打包好的这个提示词给gpt-image-2，开始生成图片。
第四步：分析深度理解生成的图片，进行预期度测评。
第五步：生成文案。（按照"公司产品数据.txt"，生成"产品介绍，买点，引导观众购买"等模板，50到100字内的文案，类似："有没有喝啤酒第二天头疼的？那是工业勾兑酒！试试这款S101苦荞精酿，纯粮发酵28天，不像普通啤酒七天就兑水上市。入口绵柔，回口荞香，好喝不上头。主页有链接"，也有相似度严查，这次生成的文案，不能跟历史生成的文案相似度超过60%）
第六步：整理所有工作，生成最终的用于生成视频的提示词。格式："《帮我生成一个视频：
（1）素材图解析：比如：图1是：.......图x是：......
（2）排版编排剧情：放第一步就生成的排版剧情原文。（比如这x个素材图要怎么怎么编排连贯转场怎么剧情发展怎么描述什么地方什么什么 （用到图片时候可艾特对应图，比如@图1中冰柜打开.....等等所有剧情编排排版剧情内容等都包含的排版编排剧情原文）
（3）文案诵读：比如：第一秒开始（男，女随机），开始人声：（这里放入刚生成的文案原文本，比如："有没有喝啤酒第二天头疼的？那是工业勾兑酒！试试这款S101苦荞精酿，纯粮发酵28天，不像普通啤酒七天就兑水上市。入口绵柔，回口荞香，好喝不上头。主页有链接。"最后一秒人声结束）》"，具体更详细更规范的请参考"D:\0000\模板视频分析结果"文件夹里的三个txt文件具体实现逻辑。
第七步：把这个提示词和生成的素材图给视频生成模型，拿到最后的视频结果。

目录下的"D:\0000\模板视频分析结果"文件夹里的三个txt文件全是一个视频的分析结果（https://v.douyin.com/b9xiPL1Z4ZQ）或者目录下的："青柠果香中和了啤酒苦涩，喝起来清爽过瘾！#啤酒_#饮料推荐__720p.mp4"的所有超详细解析，"D:\0000\模板图分析结果"中的txt文件中写的是我们的目标产品按要求生成的四张图（一张图分了四个部分那就是四个镜头四张图，）的分析结果，你也可以自己先理解提到的三种性质的视频链接，和我们本地自己的商品图片："2.png"，请你完整认真深度阅读思考得出结论与灵感，最终整理所有步骤。
