# AGENTS.md

## 工作范围

你只能做两类事情：

1. 分析代码、解释问题、给出修改方案
2. 按明确要求修改代码

## 禁止行为

除非我明确要求，否则不要：

- 运行测试
- 启动前端或后端服务
- 执行 npm / pnpm / yarn / pip install
- 执行数据库迁移
- 删除文件
- 重构无关代码
- 修改 UI 风格
- 改动配置文件
- 自动提交 git commit

## 修改原则

- 只改和当前问题直接相关的代码
- 保持现有项目结构
- 不要引入新依赖
- 不要大范围重构
- 不要“顺手优化”
- 修改前先说明你准备改哪些文件
- 修改后说明改了什么、为什么这么改

## 测试要求

默认不要执行测试、构建、启动服务。

如果你认为必须测试，只能先说明原因，并等待我明确批准。

## 输出要求

每次回答按这个格式：

1. 问题分析
2. 修改文件
3. 修改内容
4. 注意事项
